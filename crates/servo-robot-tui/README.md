# servo-robot-tui

基于 ratatui 的 STM32 机器人监控 TUI 界面，用于实时显示传感器数据、系统状态和控制命令。

## 特性

- 实时折线图显示舵机电流和电池数据
- IMU 姿态可视化
- 电池状态监控（含充电信息）
- STM32 系统信息显示
- 事件状态栏（充电、风扇、保护标志）
- 交互式命令栏（配置查询/编辑）
- 支持直接调用 Driver 或 ROS2 节点通讯

## 快速开始

```bash
# 使用 MockTransport 运行（无需硬件）
cargo run -p servo-robot-tui

# 启用 mock feature
cargo run -p servo-robot-tui --features mock
```

## 快捷键

| 按键 | 功能 | 颜色说明 |
|------|------|----------|
| R | 复位设备 | 红色（破坏性） |
| S | 关机 | 红色（破坏性） |
| 1 | 切换舵机电源 | 绿色=开/红色=关 |
| 2 | 切换 5V 电源 | 绿色=开/红色=关 |
| 3 | 切换充电 | 绿色=开/红色=关 |
| 4 | 切换电池额外输出 | 绿色=开/红色=关 |
| Q | 查询所有配置 | 蓝色 |
| W | 打开配置编辑器 | 蓝色 |
| Esc | 退出/关闭弹窗 | - |

## 架构设计

```
src/
├── main.rs                 # 入口，事件循环，按键处理
├── app.rs                  # App 状态管理
│   ├── App                 # 主状态结构体
│   ├── ConfigEditor        # 配置编辑器状态
│   ├── ConfirmDialog       # 确认对话框状态
│   └── DataSnapshot        # 数据快照（从 driver 获取）
├── data_source.rs          # 数据源抽象
│   ├── DataSource trait    # 统一数据接口
│   └── DriverSource        # 基于 Driver 的实现
└── ui/
    ├── mod.rs              # UI 渲染主入口，布局管理
    ├── power_chart.rs      # 舵机电流折线图
    ├── battery_chart.rs    # 电池折线图（含充电数据）
    ├── imu_display.rs      # IMU 姿态可视化
    ├── system_info.rs      # STM32 系统信息
    ├── battery_widget.rs   # 电池状态组件
    ├── event_bar.rs        # 事件状态栏
    ├── command_bar.rs      # 命令栏/配置编辑器
    └── colors.rs           # 颜色阈值配置
```

### 数据流

```
Driver/ROS2 → DataSource → App.update() → UI.render()
                ↓
           DataSnapshot (线程安全快照)
                ↓
           ChartData (折线图数据)
```

### 关键设计

1. **数据源抽象**：`DataSource` trait 支持 Driver 和 ROS2 两种数据源
2. **状态快照**：`DataSnapshot` 包含所有传感器数据的最新值
3. **折线图数据**：`ChartData` 使用 `VecDeque` 存储最近 N 个数据点
4. **颜色阈值**：根据配置中的电流/温度限制动态调整显示颜色
5. **命令模式**：支持确认对话框、配置查询、配置编辑三种模式
